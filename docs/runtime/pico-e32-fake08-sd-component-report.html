The SD driver becomes a configurable component

The onboard-microSD mount moved out of the fake-08 app and into a reusable, board-agnostic component. Each board now owns its SD wiring the way it already owns its display, and an owns_bus flag lets a future board share the SPI bus with its screen. Same behaviour on this board — verified on hardware — with two latent teardown bugs fixed on the way.

01 The ownership split

One rule drives the design: the reusable component knows only SPI + FATFS, never a board. Three actors, one contract — the sdcard_spi_config_t struct — split so each field has a single owner. The board fills the hardware half; the app fills the policy half; the component is the mechanism and depends on neither.

Dependencies point one way: the app and board depend on the component’s config type; the component depends on neither. Swapping boards is a boards/<board>/board.cpp swap — no app or component edit.

02 Why it needed to be configurable

The pins were hardcoded in an app file — fine for one board. But there are two boards with different SD wiring, and the difference isn’t only pin numbers: it’s who owns the SPI bus. That’s the flag the old code couldn’t express, and the reason a plain pin swap wasn’t enough.

The owns_bus flag is the whole generalization: the same mount path serves a private bus and a shared one, and the teardown frees the bus only when the component brought it up.

03 The contract: hardware vs policy

Every field of sdcard_spi_config_t has one owner. sdcard_spi_config_default() seeds the policy; board_sd_config() overwrites only the hardware. Neither steps on the other.

Hardware — board fills

  • host — SPI2_HOST
  • pin_cs / mosi / miso / sclk
  • owns_bus — private vs shared
  • max_freq_khz — drop if flaky

Policy — app fills

  • mount_point — "/sdcard"
  • format_if_mount_failed — false
  • max_files — 5
  • allocation_unit_size — 0

04 Two latent bugs, fixed in the move

The old sd_unmount() freed the bus with a hardcoded host and unconditionally — harmless on the one board that always owned SPI2, but wrong the moment a second board shares a bus, or a mount fails before the bus comes up. The component records the host it used and whether it owns the bus, and frees accordingly.

before — sd_mount.cpp
void sd_unmount(void) {
    if (!s_card) return;
    esp_vfs_fat_sdcard_unmount(SD_MOUNT_POINT, s_card);
    spi_bus_free(SPI2_HOST);   // hardcoded host, always frees
    s_card = nullptr;
}
after — components/sdcard_spi/sdcard_spi.c
void sdcard_spi_unmount(void) {
    if (!s_card) return;
    esp_vfs_fat_sdcard_unmount(s_mount_point, s_card);
    if (s_bus_owned) spi_bus_free(s_host);   // the host we used, iff we own it
    s_card = NULL;  s_bus_owned = false;
}

06 Verified on hardware

Flashed to the board (CP2104 bridge, ttyUSB1) and captured the boot log. The tell is the log tag sdcard_spi — the old app-local driver logged as sd, so the code that ran is the new component. Full round-trip: mount → scan → load a cart from SD → GameLoop.

Behaviour on this board is identical by construction — same host, pins, owns_bus=true, policy, and mount point as the loader that was already hardware-verified. The draw path is untouched, so the panel shows HELLO.P8 exactly as before.
