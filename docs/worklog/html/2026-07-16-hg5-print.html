The font: authored, not sourced

The host-graphics doc established that PICO-8's font is CC-0, and that an emulator's file (fake-08's fontdata.cpp, MIT, with no font copyright notice) shouldn't be copied — the data may be transcribed, but not the file. For HG-5 I went one step more conservative: the glyphs are authored as 3×5 row-strings in assets/pico8_font.h, so there is no third-party data at all — CC-0 or otherwise — and the whole provenance question is moot. The trade-off is fidelity: these are PICO-8-style glyphs, not pixel-identical.

📋 Backlog RT-FONT — rebuild pico8_font.h from a real PICO-8 font. The header is a hand-authored placeholder. PICO-8's own font is CC-0 (8 bytes/char, 8×8 bitfield, LSB = left pixel); export it (a PNG or a font-memory dump) and convert it — ideally via an assets/gen_font.py that emits pico8_font.h, the same pattern gen_celeste_cart.py uses for the sprite sheet — for pixel-exact glyphs with zero hand-drawing. Blocked only on obtaining the font bytes (no PICO-8 on this dev box). A separate .h makes the eventual swap a one-file change that never touches the renderer.

Each glyph is authored as five 3-char row-strings, decoded to a bitmap at init. Readable in the source, so a wrong glyph is visible and fixable.
