HG-6: pal, camera, clip — the drawing surface is complete

✓ done & verified surface complete: HG-1…HG-6
The last of the graphics surface: pal() (colour remap — Celeste's flash), camera() (offset — screen shake), clip() (bounded drawing). All three apply at the single px() chokepoint, so every op honours them; identity defaults keep every earlier test byte-identical. With this, spr+map+print+camera+pal is every draw call a real Celeste _draw makes.

One chokepoint, three effects

Semantics, matched to what Celeste actually uses (checked against celeste.p8):

  • pal(c0,c1) — the draw palette (Celeste's flash). pal() resets palette + transparency, as PICO-8 does. The p=1 screen palette isn't modelled — Celeste never uses it.
  • camera([x],[y]) — offsets drawing by (-x,-y); Celeste's screen shake.
  • clip([x,y,w,h]) — bounds drawing to a rectangle. Celeste never uses it, but it's cheap and part of the surface. (palt transparency was done in HG-3.)

Verification — numeric, then on a real room

Numeric (P8_GFXTEST=1, read from the framebuffer) — all pass:

clip inside (30,30)=green      fb= 3   (full-screen fill, clipped to an 8..55 box)
clip outside (60,60)=bg        fb= 0
pal remap 8->12 (90,18)        fb=12   (rectfill colour 8, drawn under pal(8,12))
pal control 8 (90,44)          fb= 8   (same rectfill, pal reset)
camera marker screen(68,68)    fb=10   (drawn at world 100,100 under camera(36,36))
camera: world(100,100)         fb= 0   (nothing there — it moved to screen space)

On a real Celeste room: P8_CAMX=8 P8_CAMY=6 renders the room shifted up-left (camera works on real content); P8_FLASH=1 (for i=0,15 do pal(i,7) end) renders it white-on-black — every colour remapped to 7, geometry preserved. That's PICO-8's death-flash; confirmed the only non-background colour in the frame is 7.
