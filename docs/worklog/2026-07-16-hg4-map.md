## The shared-memory problem, resolved

PICO-8's map is **128×64 cells**, but the `__map__` section carries only **rows 0–31** (4096 B). Rows
32–63 share memory with the **bottom half of the sprite sheet** (gfx rows 64–127, PICO-8 memory
`0x1000–0x1FFF`), and Celeste uses it: rooms 16–31 live there. A `map()` fed only from `__map__` renders
**half the game**.

**Generator fix** (`gen_celeste_cart.py`): emit the full 8192-byte map — rows 0–31 from `__map__`, rows
32–63 by packing gfx pixels of rows 64–127 two per byte (`tile = px[2x] | px[2x+1]<<4`, low nibble =
left pixel). The two halves' tile statistics are near-identical, which is the first sign it's real map
data, not noise:

| | zeros | tile&lt;64 | max id |
|---|---|---|---|
| rows 0–31 (`__map__`) | 36% | 96% | 104 |
| rows 32–63 (shared) | 40% | 95% | 127 |

## Gate #2 (luabench) is unaffected — the consumer check held

`CELESTE_MAP` grew 4096 → 8192, and it's shared with `pico-e32-luabench`. Safe, because luabench's
`mget`/`mset` **clamp to `y<32`** (`celeste_bench.cpp:17,22`) — they only ever index `[0,4096)`, and
`map=nop` there (it's headless). The first 4096 bytes are byte-identical (fingerprint `408a1f31`, max tile
104), and luabench builds clean against the 8192 array. So **Gate #2 sees exactly what it saw before.**
*(Its `y<32` clamp still hides rooms 16–31 from the benchmark — extending it to `y<64` and running levels
16–30 would benchmark the full game, but that's a Gate #2 re-scope, not HG-4.)*

## Runtime

`firmware/pico-e32-host/main/host_main.cpp`:

- Extracted the blit loop into `draw_sprite()`, shared by `spr()` and `map()`.
- **`map(celx,cely,sx,sy,[celw=128],[celh=64],[layer=0])`** — draws a cell block; each non-zero tile is
  drawn as a sprite (tile 0 = empty, skipped); `layer` filters on sprite flags via `fget`.
- **`mget`/`mset`** (full 128×64) and **`fget`/`fset`** — reading/poking `g_map` (128×64) and `g_gff`
  (256 flags), both null-safe so the trivial cart is unaffected.

**Trivial-cart `fb_hash` unchanged** (`5e851a75`) — `map()`/`mget`/etc. are null-safe, so registering
them changed nothing for a cart that never calls them. No Gate #3 regression.

### The `layer` filter — a verified strict subset

Rendered room 1 with each single-bit layer and compared to the full render:

```
full room 1: 10827 lit px
  layer 0x01:  6720 px   subset-of-full = True
  layer 0x02:  6869 px   subset-of-full = True
  layer 0x04:  3916 px   subset-of-full = True
  layer 0x08..0x80:  0 px   (Celeste doesn't use those flag bits)
  union of all layers: 10785 px  (vs full 10827)
```

Every layered render is a strict subset of the full one; flag bits 0–2 carry content (real `CELESTE_GFF`
data through `fget`); the union reconstructs the full render **minus 42 px** — the flagless tiles, which
PICO-8 draws only at `layer=0`. Correct behaviour, confirmed.
